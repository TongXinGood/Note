## 2.计算机的系统结构思想

### 2.1 面向摩尔定律的设计

**摩尔定律：**由Intel的创始人之一戈登·摩尔在1965年提出，它指出在一个芯片上集成的晶体管数量每18到24个月就会翻一番。

当然对于摩尔定律来说，它必定会受到一个物理限制，因为对于芯片来说，它的大小是固定的，它所能集成的晶体管数量也是有限的，所以它并不可能永久实现

### 2.2 使用抽象来简化设计

什么是抽象？抽象是一个非常重要的思想，在计算机的架构当中，包括程序的架构中.我们必须发明一个能够提升产生、提升生产效率的技术.这个技术之一就是抽象。‘

那抽象为什么能提升效率呢？抽象主要就是用来表示不同的设计层次，高层次的看不见低层次的细节，只能看见一个简化的模型.

例如对于计算机的系统设计里面，分为以下几层

1.**应用层**：这是用户直接交互的层，如浏览器、文字处理软件等.在这个层次上，我们看不到硬件的具体细节，只能看到各种应用程序.

2.**操作系统层**：这一层管理计算机的硬件资源，并为应用程序提供服务.在这一层，我们看不到硬件的具体操作，只能看到操作系统提供的接口和功能

3.**硬件驱动层**：这一层是操作系统和硬件之间的桥梁，负责控制和管理硬件设备.在这一层，我们看不到硬件的物理结构，只能看到驱动程序提供的接口.

4.**硬件层**：这是系统的最底层，包括处理器、内存、硬盘等硬件设备.在这一层，我们可以
看到物理设备的具体细节.

那通过这些分层，设计师和程序员就可以更加高效地工作，因为他们不需要同时关注所有的细节.每个层次的抽象都为上一层提供了一个简化的模型，使得设计和开发过程更加可控和可管理

### 2.3 加速大概率事件

加速大概率事件就是：实际上这是计算机科学和工程中的一个普遍原则：优化瓶颈（BottleneckOptimization）。在许多系统中，大部分的时间或资源往往被少数几个关键操作或组件所消耗。这些操作或组件就构成了系统的瓶颈，它们限制了系统的整体性能或效率，也就是说有80%的时间在干20%的事情，所以我们优化的是这些20%的关键组件。

那我们下需要先找到哪些资源是大概率的(由统计来决定)，然后把这些资源策略处理，那有哪些方法呢，如下：

**缓存：**是一种常用的优化技术，用于加速数据的访问速度。通过将频繁访问的数据存储在高速缓存中，可以减少对慢速存储设备的访问，从而提高整体性能。

例如：假设我现在经常要喝水，那这个水杯是放在我旁边好，还是放在隔壁的房间好呢，很显然，放在旁边是最好的，因为拿取速度快

**数据压缩：**在数据传输或存储时，通过压缩数据可以减少所需的空间或带宽。这有助于减少I/O操作、提高存储效率，并加速数据传输。

**预处理：**通过提前对数据进行处理或计算，可以避免在实时处理时的复杂性和延迟。预处理可以包括数据清洗、索引创建、聚合计算等

**预测和推测技术：**这些技术试图提前预测或猜测未来的操作或需求，从而提前进行准备或执行。例如，预测模型可以预测用户的下一个操作，而推测执行则可以尝试提前执行某些操作以提高响应速度。

### 2.4 通过并行来提高性能

由于现代CPU无法增加晶体管来提高性能，所以通过多核来增加处理速度，也就是人越多办事越快

并发：一个时间段内多个任务同时响应

并行：也就是上面的多核处理

### 2.5 通过流水线来提高性能

很简单，就类似于工厂一样，一个工人只负责一个模块的处理，然后像流水线一样向下个工人来处理技术，以此类推，现代CPU 都有了这种流水技术，来提高性能也就是并行性能

例如，在处理器设计中，指令执行通常可以分为取指（Fetch）、译码（Decode）、执行（Execute）、访存（Memory Access）、写回（Write Back）等多个阶段。通过流水线技术，不同的指令可以在不同的流水线阶段同时进行处理，每个阶段都有专门的硬件单元来理，从而实现并行执行。

但是流水线是会被打断的，效率降低（分支、函数调用、异常[syscalll]）,这个后面会具体说

### 2.6 通过预测来提高性能

预测技术是计算机系统结构设计中提高性能的重要手段之一.预测技术通过预测未来的指令或数据，提前进行预取和预处理，从而减少等待时间和提高处理速度.预测技术通常应用于多个领域，包括指令预测、分支预测、缓存预取等。

**指令预测：**指令预测是一种预测技术，用于预测下一个要执行的指令。处理器可以基于之前的指令序列来预测未来的指令，并提前进行解码和准备。这样，当实际的指令到达时，处理器可以更快地执行它，从而提高执行速度。

**分支预测：**分支预测是另一种常见的预测技术，用于预测条件分支指令的结果。处理器会尝试预测分支指令是否会被执行（即条件为真或假），并提前加载和执行相应的指令。这样，当分支指令的结果确定时，处理器已经准备好了要执行的指令，从而避免了延迟

**缓存预取：**缓存预取是一种预测技术，用于预测哪些数据将被访问，并提前将这些数据加载到缓存中。处理器可以基于之前的访问模式来预测未来的数据访问，并提前从内存中加载这些数据到缓存中。当实际访问这些数据时，它们已经位于缓存中，从而提高了访问速度。

### 2.7 存储器层次结构

我们知道CPU的速度是快的，存储器，硬盘的效率是慢的.而我们的数据无时无刻不在进行着读取操作.所以，存储器速度通常就影响着性能.存储的大小规模就决定了他能处理问题的规模大小，速度越快，容量越小，价格越贵；速度越慢，容量越打，价格越便宜；

那对于寄存器的层次结构是什么呢：

**最顶层：**通常是**寄存器（Registers）或者缓存（Cache）**，它们速度最快，但容量最小，且单位容量的成本最高。CPU可以直接访问寄存器，而高速缓存则用来临时存储最近频繁使用的数据或指令以减少主存访问次数。

**主存（RAM，Random Access Memory）：**其速度相较于寄存器和缓存慢，但容量大得多，价格也相应降低。主存是CPU直接寻址和交换数据的主要区域。

再下一层可能包括**虚拟内存（Virtual Memory）、磁盘缓存（Disk Cache）**等，它们的速度进一步下降，但提供更大的存储空间，同时价格相对更低。

**底层**则是硬盘**驱动器（HDD）和固态硬盘（SSD）**等永久性存储介质，它们的访问速度最慢，但提供了TB级别的大容量存储，并且单位存储成本最低。

### 2.8 通过冗余提高可靠性

冗余技术就是：通过增加额外的硬件组件来提高系统的可靠性和容错能力

**处理器冗余：**使用多核处理器或者构建集群系统，当一个核心或节点失效时，其他部分可以接管任务，保证服务不中断。例如，云计算平台中的服务器集群就广泛采用了这种策略。

**存储冗余：**RAID（Redundant Array of Independent Disks）技术是一种常见的存储冗余方法，通过磁盘阵列将数据分布在多个硬盘上，实现数据备份、错误校验以及在单个硬盘故障时仍能正常读写数据

**网络冗余：**在网络设备中设置备用路径或备用设备，如负载均衡器、交换机的链路聚合（Link Aggregation）、路由协议中的热备路径等，当主路径出现问题时，流量可以迅速切换到备份路径或设备。

**电源冗余：**数据中心通常会采用双路供电或多路供电系统，确保一路电源故障时另一路电源能够继续为服务器和其他关键设备供电。

**冷却冗余：**大型数据中心还会对冷却系统进行冗余设计，以防止因冷却系统故障导致服务器过热而宕机。

#### 5.2.1 CPU的时钟频率、时钟周期

CPU的时钟（震荡波形来驱动CPU做事情，时钟驱动时序，时许驱动事件）

时钟的单位是赫兹（Hz），常对应的是1Hz 1KHz 1MHz 1GHz 1THz

周期的单位就是秒，那与上面对应单位 1s    1ms   1us     1ns 

PS:每一个单位都对应10的三次方

#### 5.2.2 CPI

指的是cycles per instruction,**执行一条指令所需要的时钟周期数量**，单位是次数，也就是需要多少个周期

比如：一个乘法指令需要5个周期，一个加法指令，需要1个周期，所以不同的指令，执行周期不一样

一个程序的CPI:构成该程序的所有指令需要的时钟周期数的平均数

CPI=程序执行所需时钟周期数/程序包含的指令数量

#### 5.2.3 CPU执行时间

CPU执行时间=（程序所需要的指令数量*CPI）×时钟周期

周期数*每个周期占用时间

由于时钟频率和时钟周期时间互为倒数关系，上述公式也可以表示为：
CPU执行时间 = CPU时钟周期数 / 时钟频率

#### 5.2.4 IPC

Instruction per cycles **每个时钟周期能执行的指令数量**

随着指令流水线、多核的发展，IPC现在可以大于1

#### 5.2.5 MIPS

**每秒执行多少百万条指令**

MIPS=(程序所包含的指令数量➗CPU的执行时间)➗10^-6

常见单位缩写：

- K/M/G/T/P/E/Z(K为10^3,后面每一个都是前面的10^3倍)

为了方便理解冯诺伊曼体系结构，我们需要了解一下一点计算机历史，在1945年左右，一帮科学家做出了一个机器叫做ENIAC,然后有一个抽象的概念：**作业任务**，这个作业任务指的是把我们要做的事情放到一个开关电路上面然后作为一个模子送到计算机里面工作，但是这个东西只能做专门一个功能，每换一个功能就要重新设计去接线

#### 7.3.3 大小端存储

有了编址，编写地址，也就要有寻址。也就是寻找地址。首先，既然编址按字来，那么寻址也是按字来。有两种类型的字节寻址的计算机：大端和小端。

1. 大端（Big-endian）：在大端编址方式中，一个多字节数据的高位字节存储在内存的低地址处，而低位字节则存储在内存的高地址处。例如，一个32位的字（假设其值为0x12345678）在内存中会被存储为：

   ```
   地址: 0x0000 0x12
   地址: 0x0001 0x34
   地址: 0x0002 0x56
   地址: 0x0003 0x78
   ```

   MIPS 架构就是使用大端编址的。

2. 小端（Little-endian）：在小端编址方式中，一个多字节数据的低位字节存储在内存的低地址处，而高位字节则存储在内存的高地址处。对于同样的32位字（0x12345678），它在内存中的存储方式会是：

```
地址: 0x0000 0x78
地址: 0x0001 0x56
地址: 0x0002 0x34
地址: 0x0003 0x12
```

​	许多现代的PC和个人设备（如Intel和AMD的x86和x64架构）使用小端编址。

#### 7.6.1 小知识

二进制、八进制、十进制、十六进制本身都是同一个东西，表示的数字都是一样的

我们拿123举例子，我们可以看成是100+20+3，那也就是1*10^2+2×10^1+3×10^0,那对于二进制的10000，也就是1×2^4

那么0x1和1是一个东西，那么对于0x1<<10,也就是1×2^10次方

但是后来我们发现一次性写20个零太多了，比如二进制里面的0x1<<20,所以根据二进制和十六进制的转换，每四个二进制可以表示一个十六进制，所以我们只需要写出五个十六进制的0即可（上述所有的移位都是二进制）

0xF101 === 1111 0001 0000 0001

0x代表在计算机里面后面就是16进制，0x没有任何意义

#### 7.6.2 有符号和无符号数

那我们回归正题，我们来想想怎么表示任意的数字，数字的表示有进制表示法，本质就是位权求和表示法，二进制、八进制、十进制、十六进制都是使用位权求和表示法，只要用这个方法表示我们就称为**无符号数**

那么如下图：

![](./COA/C40.png)

那么5的位置我们一般称为位权，无论X是多少，那个位置都是X×base^5,往后推则是X×base^4,那现在就有一个新的问题就是我怎么表示负数呢，于是科学家想出来，让第一个位置变成-1×base^5,如下图：

![](./COA/C41.png)

于是科学家就提出了一个新的概念，让最高位的权位设置为-1×权位值吗，其他位为正数，还是做位权求和，这个得到的数字就叫做**有符号数**

然后我们发现 1111 == -8+4+2+1 = -1，然后我们就会发现任何一个数A，与它的反数（每一位取反）他们的和都是1111

也就是A+~A 位数全是1，然后我们就发现A+~A + 1 =0

那我们再来看一下什么是补数的概念，我们以钟表为例子，假设现在时间是6点，我们想要它往回走，到2点的位置，那是不是6-4=2；那如果我们只能往前走，也就是加，那是不是6+8=14%12=2，我们发现也是2点，那么这时候我们称-4和8互为补数

于是根据上面的我们就定义上述的 ~A+1为A的补数，也就是A的补码，那所谓的-A也就是~A+1,其实计算机就是一个钟表，越界太多，就相当于往回走
